Este endpoint utiliza OpenAI para analizar y corregir autom√°ticamente las multiplicidades en diagramas GoJS siguiendo la l√≥gica UML correcta.

## üéØ Endpoint

## üìù Funci√≥n

Toma un JSON de GoJS existente y corrige √∫nicamente las multiplicidades (`fromMultiplicity` y `toMultiplicity`) en el `linkDataArray`, siguiendo estas reglas:

### ‚úÖ Reglas de Correcci√≥n

#### 1. **Composici√≥n** (rombo negro - "es parte de")
- **L√≥gica**: Las partes pertenecen al todo
- **Ejemplo**: Habitacion es parte de Edificio
- **Correcci√≥n**: `from=Habitacion, to=Edificio, fromMultiplicity="*", toMultiplicity="1"`

#### 2. **Agregaci√≥n** (rombo blanco - "tiene/contiene") 
- **L√≥gica**: El todo tiene partes, pero pueden existir independientemente
- **Ejemplo**: Departamento tiene Empleados
- **Correcci√≥n**: `from=Empleado, to=Departamento, fromMultiplicity="*", toMultiplicity="1"`

#### 3. **Herencia/Generalizaci√≥n** (flecha vac√≠a)
- **L√≥gica**: Siempre relaci√≥n de herencia
- **Ejemplo**: Perro hereda de Animal
- **Correcci√≥n**: `from=Perro, to=Animal, fromMultiplicity="*", toMultiplicity="1"`

#### 4. **Asociaci√≥n** (l√≠nea simple)
- **L√≥gica**: Analiza el contexto y nombres para determinar la cardinalidad
- **Ejemplos**:
  - Cliente-Pedido: `from=Pedido, to=Cliente, "*" a "1"`
  - Usuario-Perfil: `from=Usuario, to=Perfil, "1" a "1"`
  - Estudiante-Curso: `from=Estudiante, to=Curso, "*" a "*"`

### Hook de React

```javascript
import { useState } from 'react';

export function useMultiplicityFixer() {
  const [loading, setLoading] = useState(false);
  const [result, setResult] = useState(null);
  const [error, setError] = useState(null);

  const fixMultiplicity = async (diagram) => {
    setLoading(true);
    setError(null);

    try {
      const response = await fetch('http://localhost:3001/ai/fix-multiplicity', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ gojsDiagram: diagram })
      });

      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);

      const data = await response.json();
      setResult(data);
      return data.correctedDiagram;
    } catch (err) {
      setError(err.message);
      throw err;
    } finally {
      setLoading(false);
    }
  };

  return { fixMultiplicity, loading, result, error };
}
```

- **`from/to`**: Cambios en multiplicidades origen y destino

## ‚ö†Ô∏è Limitaciones y Consideraciones

### ‚úÖ Lo que S√ç hace:
- Corrige multiplicidades siguiendo l√≥gica UML
- Mantiene toda la estructura original del JSON
- Proporciona an√°lisis de cambios realizados
- Valida que el JSON de salida sea v√°lido

### ‚ùå Lo que NO hace:
- No cambia nombres de clases o atributos
- No modifica coordenadas (loc) de elementos
- No altera las categor√≠as de relaci√≥n
- No agrega o elimina nodos/links

### üéØ Precisi√≥n:
- **Alta** para relaciones est√°ndar (Edificio-Habitacion, Cliente-Pedido)
- **Media** para relaciones ambiguas que requieren contexto de negocio
- **Muy Alta** para herencia (siempre * a 1)
